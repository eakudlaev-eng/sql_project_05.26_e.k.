# Информация о проекте

Это небольшой SQL-проект про эффективность CRM-коммуникаций на синтетических банковских данных. Я делал его как приближенную к рабочей задачу: есть клиентская база, несколько кампаний, test/control группы, сообщения, события по коммуникациям и заявки после контакта с клиентом.

Что я хотел проверить:

- какие кампании дают больше заявок и одобрений
- какой канал работает лучше: push, SMS или email
- какие клиентские сегменты лучше реагируют на коммуникации
- есть ли эффект относительно контрольной группы
- кого можно отобрать для повторной коммуникации

В данных: 3000 клиентов, 4 CRM-кампании, 3756 записей в аудиториях кампаний, 3396 сообщений, 8216 событий коммуникаций и 202 заявки.

Главный вывод: по абсолютному числу одобрений лучше всего сработало повторное предложение кредита наличными. Эта же кампания дала самый сильный uplift относительно контрольной группы: +2,26 п.п. По сегментам наиболее стабильно выглядит `salary`: он входит в топ по конверсии в нескольких кредитных кампаниях.

В проекте лежат CSV-файлы с данными, SQL-схема, SQL-запросы анализа и короткий аналитический отчёт. Данные синтетические, но логика похожа на реальную задачу CRM/аналитика данных: проверить источники, собрать воронку коммуникаций, сравнить кампании и подготовить аудиторию для ретаргетинга.

Файлы:

- `data/` — исходные CSV-таблицы;
- `sql/schema.sql` — структура базы;
- `sql/analysis.sql` — запросы для анализа;
- `Аналитический_отчет.md` — краткие выводы;
- `Справочник_данных.md` — описание таблиц и полей.
